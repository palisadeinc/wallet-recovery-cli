# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please email **bugs@ripple.com** with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

We will acknowledge receipt within 48 hours and aim to provide a fix within 30 days for critical issues.

## Security Best Practices

When using this tool:

- Store recovery keypairs securely (encrypted, offline if possible)
- Use strong passwords (minimum 8 characters) for encrypted keys
- Never share private keys or recovery data
- Verify checksums when downloading release binaries
