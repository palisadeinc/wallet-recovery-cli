# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities to security@palisade.co

Do NOT create public GitHub issues for security vulnerabilities.

We will acknowledge receipt of your vulnerability report within 48 hours and will send a more detailed response within 7 days indicating the next steps in handling your report.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Security Best Practices

When using this tool:
- Store RSA private keys securely with restricted file permissions (0400)
- Use strong passwords when encrypting recovered keys
- Keep recovery kit files secure and delete them after use
- Verify checksums of downloaded binaries before use
